One particular run of the "sudoku_bench" binary:

Each implementation is run ITER (currently at 100) times and the average between these runs is displayed.
The unit of measurement is the nanoseconds.
For each board, the measurements are done in two different ways:
1) The total time spent to solve ONLY a number of ITER boards, divided by iter -> The time spent to solve ONLY one board
2) The time spent to read the board, check its validity and to solve it. 
It is run ITER times and the resulting time amount is divided by ITER.
Obviosly, this number will be higher considering the same implementation for the same board.
